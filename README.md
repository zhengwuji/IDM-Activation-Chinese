## 🌟 项目速览

**IDM 激活脚本中文版** 是面向中文 Windows 用户定制的 IDM 试用与激活管理工具。为了解决原版英文脚本在中文控制台（CP936/GBK）下乱码、新手不会提权、杀软误报无从排查等实际痛点而生。

本项目的所有脚本逻辑均开源可见，并且我们引入了严格的 CI/CD 静态校验和自动化构建流程，每次推送代码都会自动打包、生成哈希并发布 Release，确保您下载的每一个版本都是安全可审查的。

### 📦 核心入口文件

| 核心文件 | 用途与运行机制 |
| --- | --- |
| `开始激活.cmd` | **新手唯一需要运行的主文件**。双击将自动请求管理员权限（自带 VBScript 双轨兜底提权） → 执行多项极速环境自检 → 弹出中文交互式主菜单。 |
| `IAS.cmd` | 核心执行引擎，供 `开始激活.cmd` 后台调用；高级用户亦可在管理员终端通过命令行参数静默执行。 |
| `使用说明.txt` | 极简新手上手说明，采用 UTF-8 BOM 编码，防止在现代 Windows 记事本中打开时乱码。 |

## ✨ 主要功能特性

* ✅ **三种激活模式**
  1. **[1] 激活（冻结）**：锁定 IDM 当前的 30 天试用期。不写序列号，最不易被 IDM 服务器判定为“假序列号”，最适合正在使用官方试用期的用户或兜底激活。
  2. **[2] 激活（推荐）**：一键注入随机生成的授权注册信息，激活后立即可用，无需提前注册账号或开始试用。
  3. **[3] 重置激活/试用期**：一键彻底清除注册表内残存的各种 IDM 激活及过期锁键，使 IDM 恢复到最初安装的纯净未激活状态。
* ✅ **极速环境自检**
  - 开始激活前，将一次性执行：管理员权限、IAS.cmd 存在性、PowerShell 语言模式、系统 Null 服务状态、网络连通性、控制台代码页（GBK）、WMI/CIM 服务状态、IDM 注册表与便携版路径、以及当前目录写权限等 10 项自检，并在界面上实时标出有问题的项。
* ✅ **UAC 提权双轨兜底**
  - 自动申请管理员权限。在系统组策略对 PowerShell 进行强行锁定（非 FullLanguage 模式或 AppLocker）时，脚本将自动切换到临时 VBScript 执行提权，极大地提升了脚本在企业或学校电脑上的兼容性。
* ✅ **绿色/便携版路径自动探测**
  - 不仅扫描 HKLM/HKCU 注册表，还支持在脚本同级目录中探测 `IDMan.exe`。如果存在，自动作为安装目录，无需写入注册表即可完成便携版的激活。
* ✅ **自动注册表备份**
  - 在修改任何注册表键值前，会自动备份相关 CLSID 到 `C:\Windows\Temp\_Backup_HKCU_CLSID_[时间戳].reg`，双击备份文件即可一键恢复，安全无忧。

## 🚀 快速上手使用教程

### 第一步：解压文件
下载 ZIP 压缩包后，**请务必先将文件全部解压**到本地目录。切勿在压缩包文件查看器中直接双击运行！

### 第二步：运行主脚本
双击运行 **`开始激活.cmd`**，在弹出的用户账户控制（UAC）窗口中选择“**是**”。

### 第三步：根据检测结果与选项选择
等待极速环境自检完成（合并 PowerShell 调用后仅需 1 秒即可完成检测）。通过后，您将看到交互主菜单：

```text
┌─────────────────────────────────────┐
│   IDM 激活脚本主菜单                 │
│                                     │
│  [1] 激活（冻结）                    │
│  [2] 激活            ⭐ 推荐         │
│  [3] 重置激活/试用期                 │
│  [4] 下载 IDM                       │
│  [5] 帮助                           │
│  [0] 退出                           │
└─────────────────────────────────────┘
```

* **如何选择激活模式？**
  * 🌟 **情况一：直接可用 / 没领过试用期** ── 直接按 **`2`**（普通注册激活），一键写注册表，最省心。
  * ❄️ **情况二：已领了试用期且在使用中** ── 选 **`1`**（冻结试用期），固定当前的 30 天试用天数，最稳定。
  * 🔄 **情况三：之前提示假序列号或弹窗未注册** ── 先选 **`3`**（重置），重置完成后，再尝试选 **`2`**。如果仍提示假序列号，则再次重置后，点官方开始 30 天试用，然后选 **`1`** 进行冻结兜底。

## 🛠️ 高级命令行参数（静默安装）

系统管理员及高级用户可以直接在管理员 CMD 窗口中通过命令静默执行 `IAS.cmd`：

```cmd
# 1. 注册激活（推荐，直接可用）
IAS.cmd /act

# 2. 冻结激活（锁定当前试用期）
IAS.cmd /frz

# 3. 重置试用期/激活状态
IAS.cmd /res

# 4. 无人值守静默运行 + 输出运行日志到指定路径
IAS.cmd /act /silent /log="C:\Temp\ias_run.log"
```

## ⚠️ 注意事项与常见问题自愈

#### 1. Windows SmartScreen 提示“已保护您的电脑”？
* **原因**：这是 Windows 针对一切未数字签名的 `.cmd` 脚本的默认保护行为。
* **解决**：点击弹窗中的“**更多信息**” ─ 选择“**仍要运行**”。或者右键点击 `开始激活.cmd` ─ 选择“**属性**” ─ 勾选底部的“**解除锁定**”并点击确定。

#### 2. 贴身安全软件（Defender / 火绒 / 360）报毒拦截？
* **原因**：由于激活流程涉及向 Windows 注册表敏感项写入键值以及静默申请管理员提权，这会触发启发式杀毒引擎的敏感拦截。
* **解决**：脚本完全开源，无任何恶意二进制混淆。您可以放心将解压目录加入杀毒软件的排除项中运行。

#### 3. 提示“未在注册表找到 IDM 安装路径”？
* **原因**：您可能使用的是绿色便携版 IDM，系统注册表内未写入默认路径。
* **解决**：请直接将解压出的激活脚本（`开始激活.cmd` 和 `IAS.cmd` 等）**拷贝至 IDM 的主程序目录（即包含 `IDMan.exe` 的目录）**下，然后重新双击 `开始激活.cmd` 运行，脚本会自动从同级目录识别路径。

## ⚙️ CI/CD 自动化构建与发布流程

本项目集成了严密的持续集成与持续发布（CI/CD）管线：

1. **静态代码质检（CI）**：每次提交代码或提交 PR，GitHub Actions 会在 Windows Runner 上运行 [validate.ps1](file:///d:/Personal/Desktop/ssh/IDM-Activation-Script-Chinese/tools/validate.ps1)。该脚本将强行检测所有 `.cmd` 文件是否为 GBK 编码（防止乱码）和 CRLF 换行；并利用静态规则检测有无拼写错的 `goto/call` 标签，以及是否存在括弧配对不闭合，从源头扼杀 Regression。
2. **自动编译发布（CD）**：配置在 `.github/workflows/release.yml`。每次 Push 到主分支检测到 `IAS.cmd` 内的 `@set iasver` 发生变更时，自动进行全套打包，计算 SHA256，从 `CHANGELOG.md` 中按版本号提取更新日志，自动创建对应的 Git Tag，并将打包资产秒级推送到 GitHub Releases。

## 📝 更新日志

### v1.3.9 ─ 2026-06-24（当前版本）
* **极速环境自检**：合并 3 次独立 PowerShell 调用为单次整合执行，使自检耗时缩短近 3 倍，提高启动响应。
* **网络连接超时控制**：在网络探测时引入 1.5 秒超时控制，避免在无网或代理网络故障时长时间挂起。
* **便携版/绿色版路径探测**：支持在脚本同级目录探测 `IDMan.exe`。若检测到，将直接作为 IDM 路径，极大地地方便了便携版用户。
* **UAC 提权双轨兜底**：在 PowerShell 提权受限或失效的极端加固系统下，自动通过生成临时 VBScript 执行提权，增强提权容错性。
* **自检结构优化**：利用原生的 `%errorlevel%` 判断代替 `&&` 与 `||` 条件分支，防止语句块内 exit code 状态穿透。
* **跳转标签检测**：在 `validate.ps1` 校验器中引入 `Validate-CmdLabels`，静态检查 `goto` / `call :` 跳转的标签是否存在，防止拼写引起语法崩溃。
* **括号平衡配对检测**：在 `validate.ps1` 中加入 `Validate-Parentheses` 校验，静态分析括号的闭合状态，排除 CMD 引擎最敏感的括号语法异常。
* **编码与 CI 兼容**：为 `validate.ps1` 脚本添加强制 UTF-8 解析环境，确保在非 UTF-8 语言环境 the 本地主机上运行时不会由于 Git 包含中文文件名而引发路径异常。
* **自动化构建发布（CI/CD）**：新建 `.github/workflows/release.yml` 自动化发布动作，在 Push 到主分支检测到版本号变更时自动进行语法自检、代码打包、SHA256 生成并发布至 GitHub Releases，同时自动提取对应的 CHANGELOG 更新日志。
